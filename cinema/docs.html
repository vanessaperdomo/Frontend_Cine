Guía completa: cómo ejecutar el proyecto, glosario de términos y endpoints disponibles.

Cómo ejecutar el proyecto

1
Base de datos — levantar Docker

Entrar a la carpeta BaseDeDatos_Cine/cinema y ejecutar:

docker-compose up

Esto levanta PostgreSQL en el puerto 5433, crea las tablas automáticamente con Liquibase e inserta los datos iniciales. No se necesita hacer nada más.

2
Backend — levantar la API

Entrar a la carpeta Backend_API_Cine/cinema y ejecutar:

mvn spring-boot:run

La API quedará disponible en http://localhost:8080. Se necesita tener instalado Java 17 y Maven.

3
Frontend — abrir en el navegador

Entrar a la carpeta Frontend_Cine/cinema y abrir el archivo:

index.html

Se puede abrir con doble clic. No necesita servidor. Funciona directamente en el navegador.

4
(Opcional) Swagger — documentación interactiva de la API

Con la API corriendo, abrir en el navegador:

http://localhost:8080/swagger-ui.html

Arquitectura del sistema

FrontendHTML + CSS + JS
→ HTTP fetch →
BackendSpring Boot :8080
→ JPA →
Base de datosPostgreSQL :5433

El Frontend se comunica con el Backend a través de peticiones HTTP usando la Fetch API de JavaScript. El Backend transforma esas peticiones en consultas SQL que ejecuta en PostgreSQL a través de JPA (Java Persistence API).

Principios aplicados en el backend

PrincipioQué significaCómo se aplica en este proyecto
S — Single ResponsibilityCada clase tiene una sola responsabilidadController recibe peticiones, Service ejecuta lógica, Repository accede a datos — cada uno hace solo su parte
O — Open/ClosedAbierto para extender, cerrado para modificarSe puede agregar una nueva entidad sin tocar las existentes
L — Liskov SubstitutionLas implementaciones pueden reemplazar a sus interfacesMovieService es una interfaz; MovieServiceImpl es la implementación intercambiable
I — Interface SegregationInterfaces específicas, no genéricas gigantesCada entidad tiene su propio Service e interfaz separados
D — Dependency InversionDepender de abstracciones, no de implementacionesLos controllers dependen de la interfaz Service, no de la clase concreta
